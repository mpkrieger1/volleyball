# VCD v1.0 Post-Mortem

**Release date:** 2026-05-02 (planned)
**Build window:** 27 sprints across ~12 months (~2026-04 → 2026-05)
**Author:** Matt Krieger (solo dev)

---

## What shipped

A complete single-player NCAA Division I Women's indoor volleyball coach
dynasty simulation, Windows-first Electron app:

- **Full league:** 360 D-I programs, conferences, full 13-week regular
  season + non-conference play, conference tournaments, 64-team NCAA
  bracket with regional seeding and Final Four.
- **Sim engine:** rally-by-rally FSM with rotation, libero rules, system
  selection (5-1 / 6-2), timeouts, momentum, deterministic seeded RNG,
  and a baseline coach-AI archetype.
- **Roster + staff:** ~12 players per team across class years, position
  archetypes, ratings, redshirts; head coach + 5 assistants per team
  with hire/fire/contract mechanics.
- **Recruiting + portal + NIL:** weekly recruiting cycle with action
  budget, board management, commit resolution, Sprint 25 board-scoring
  fix that sustains rosters across multi-season dynasties; transfer
  portal entry/browsing/signing; NIL deals with cap math.
- **Postseason:** RPI computation, conference auto-bids, single-elim
  brackets, Final Four cross-region pairings, championship.
- **Awards:** AVCA All-American selection (28 honorees per season:
  2 OH / 2 MB / 1 OPP / 1 S / 1 L per team × 4 teams).
- **Analytics:** per-match charts (rotation efficiency, serve heatmap,
  rally-duration bars), per-team season stats, conference standings,
  RPI top-25, individual stat leaders.
- **UX:** Season Hub dashboard as default landing post-pick, Match Hub
  with paced PBP replay + paused-replay coach controls (timeouts,
  substitution scaffold), first-run welcome modal, Season Rhythm
  playbook modal, settings (font size, diagnostics, font scaling).
- **Engineering:** signed Windows installer, electron-updater
  auto-update, Diagnostics opt-in crash recorder, save-file
  forward-compat migration tracking, axe-core a11y validated on
  every user-facing screen.

833+ tests across unit, integration, golden-fixture, calibration, and
e2e suites. `npm run build` succeeds, `npm test` passes, `npm run
build:installer:signed` produces a signed `.exe` from the cert bundle.

## What slipped

### Per-day advance (Sprint 27 Tasks 27.4 + 27.6)

Sprint 27 spec'd a `Season.currentDate` schema column + `advanceDay` IPC
that would replace the per-week advance cadence with per-calendar-day
granularity. The user identified this gap in dev-mode testing — the per-
week advance bundles ~170 matches into one click, which feels flat.

Deferred at the end of Sprint 27 because:

- Schema migration with calibration-regen risk in a ship sprint is the
  exact kind of change CLAUDE.md §3 says to do as a dedicated commit,
  not as part of a feature mix.
- The Sprint 27 plan estimated 6–8h plus calibration verification time
  (full-season calibration suite ~10 min × multiple iterations).
- The user-experienced "see your team's match for today" need is met
  by Sprint 27 Task 27.2 (Match Hub locked to user-team match list);
  the missing piece is just the advance cadence, not the visibility.

**v1.1 path:** dedicated 1-week sprint to land the schema migration +
golden regen + IPC migration. Estimated effort: 1.5 days code + 0.5
day calibration verification.

### `TeamSeasonSummary` save-file aggregation (Sprint 25 Task 25.2)

PRD §3.5 calls for ≤25 MB save at 10 seasons. Reality: 50 MB. Sprint 24
relaxed the test bar to 60 MB; Sprint 25 deferred the schema-summary
aggregation that would close the gap.

**v1.1 path:** new `TeamSeasonSummary` Prisma table; `runOffseason`
materializes one row per team per completed season; `pruneOldSeasons`
deletes the underlying Match/Set/PMS rows for non-current seasons.
Estimated savings: ~25 MB at 10 seasons → comfortably back under the
PRD bar.

### True match-affecting in-match coach input

Sprint 26 Task 26.2 + 26.6 shipped paused-replay coach controls
(timeout button, substitution scaffold) but they're **cosmetic only** —
the sim has already played out by the time the user pauses to react.
True in-match coach input would require pausing the sim mid-rally,
applying the user's decision, and resuming the sim with the new state.

PRD §2 explicitly puts the user in the "program builder" role, not the
"in-match coach" role, so this is a v2 feature rather than a v1 gap.

### Multi-archetype coach AI

v1.0 ships with one baseline coach AI archetype. Sprint 26 plan
considered shipping aggressive / conservative / balanced variants but
deferred to v1.1.

### Deferred features (v2 backlog per PRD §2 scope)

- **Hall of Fame / all-time leaders / records book** — schema shape was
  considered (FCCD has these) but explicitly out of v1 scope.
- **Men's volleyball, beach, D-II, multiplayer, conference realignment**
  — all v2.
- **Mac / Linux builds** — v1.0 is Windows-first.
- **i18n / localization** — English-only.

### Recurring Monte Carlo flakes

Sprint 25 widened thresholds on three flakes (Sprint 9 poll, Sprint 13
recruiting top-5, Sprint 17 coaching) that recurred 6+ sprints. They're
stable now under the wider thresholds but the underlying root cause —
single-cycle Monte Carlo on zero-inflated distributions — wasn't
addressed. v1.1 candidate: pool 3 cycles before each Welch t-test.

## What to do differently for v1.1

1. **Dev-mode play-throughs at sprint boundaries.** Sprint 27's mid-sprint
   pivot to author a structural-fix sprint came from the user playing
   the build and surfacing 5 gameplay-loop gaps that no individual
   feature sprint would have caught. Schedule a dev-mode play-through
   at the end of every 4–5 sprint block, not just at release prep.
2. **Schema-summary table earlier.** The save-file budget was a known
   risk since Sprint 18 (PMS landed); Sprint 25 spec'd the fix and
   deferred. Doing it concurrent with PMS would have been cheaper than
   doing it under v1.0 ship pressure.
3. **Calibration discipline gate before threshold widening.** Three
   recurring flakes were widened in Sprint 25 instead of root-caused.
   The widenings stabilized CI but mask future regressions. v1.1
   should pool cycles or move to magnitude-ratio assertions per the
   Sprint 17 retro pattern.
4. **More integration tests for cross-cutting flows.** Sprint 23's
   dynasty test surfaced 4 latent bugs at once (PMS FK violation,
   hardcoded calendar year, missing recruit→Player promotion, save-
   size reality check). Earlier multi-cycle integration coverage would
   have surfaced these in their owning sprints.
5. **CLAUDE.md gotcha discipline applied consistently.** Sprint 14, 21,
   22, 26 retros were each filed late or backfilled. The CLAUDE.md
   "From Sprint N" gotcha block is the project's institutional
   memory — write it inline with the retro, not after.
6. **Cert provisioning earlier.** Code-signing cert was carry-forward
   from Sprint 24 → 25 → 26 → 27 because it requires a real-world
   procurement step. Resolve before the ship sprint, not during.

## v2 backlog (prioritized)

**Tier 1 — UX completion of v1 vision:**

1. Per-day advance + `Season.currentDate` (Sprint 27 deferred).
2. `TeamSeasonSummary` save-file aggregation (Sprint 25 deferred).
3. True in-match coach input (timeouts that affect outcome; subs that
   re-run the sim).
4. Multi-archetype coach AI (aggressive / conservative / balanced).
5. Hall of Fame / records book.

**Tier 2 — Realism / depth:**

6. Mid-season weekly RPI snapshots (so RPI tab works pre-postseason).
7. Substitution rules + libero entry/exit tracking in the sim.
8. Challenge calls (NCAA rule modeling).
9. Multi-cycle calibration pooling for Monte Carlo flakes.
10. Real bench list in Match Hub (Task 27.2 + 26.6 carry-forward).

**Tier 3 — Scope expansion (v2 proper):**

11. Men's volleyball.
12. Beach volleyball.
13. D-II.
14. Conference realignment over time.
15. Multiplayer.

**Tier 4 — Platform:**

16. macOS build.
17. Linux build.
18. Localization (start with Spanish/French given international
    volleyball reach).
